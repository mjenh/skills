- (decision) AD-1 Paradigm: layered with internal/ packages; Transport interface lives at the internal boundary (not public API); prevents transport details leaking into public surface while keeping testability via interface injection
- (decision) AD-2 Package split: internal/transport (Transport interface + KLAP/legacy impls), internal/crypto (shared crypto helpers — AES, key derivation, hashing); prevents duplicated crypto code across transport implementations and keeps transport focused on protocol flow
- (decision) AD-3 Transport interface: thin request/response — Login(ctx, creds) error + Send(ctx, method, payload) (response, error); Plug builds command JSON, transport handles encryption/session/wire; prevents command-building duplication across transport implementations and keeps interface stable as commands grow
- (decision) AD-4 Session ownership: transport-owned — each transport impl holds its own session state (cookies, keys, counters) internally; Plug knows login status and triggers re-auth on error, but session data is opaque; prevents forcing a common session shape across structurally different protocols
- (decision) AD-5 Concurrency: sync.Mutex at both layers — transport impl guards its own session state, Plug gates re-auth so only one goroutine re-logins on concurrent 9999 errors; zero external deps, sufficient for single-host client; prevents race conditions and redundant login storms
- (decision) AD-6 Error design: package-level sentinel vars (ErrAuth, ErrTimeout, ErrUnsupportedModel, ErrHandshake) wrapped with context via fmt.Errorf %w; callers use errors.Is; typed error structs deferred to minor version if structured fields needed; prevents over-engineering the error surface while satisfying spec's four-category discrimination
- (decision) AD-7 Transport negotiation: lazy on first command — NewPlug returns immediately with no I/O; first Send triggers KLAP attempt, falls back to legacy on protocol failure, caches the winner; Option override skips negotiation and sets transport at construction; prevents coupling construction to network and matches PRD 'login on first command' design
- (decision) AD-8 DeviceInfo decoding: lives in root package — Plug.DeviceInfo() deserializes JSON and decodes base64 fields (Nickname, SSID) in one pass into the public DeviceInfo struct; no separate internal/device package; prevents unnecessary package for a handful of lines tied to a public type
- (version) Go latest stable: 1.26.4 (June 2026); spec minimum 1.22+ confirmed reasonable; crypto/aes, crypto/cipher, crypto/rsa, crypto/md5, crypto/sha1 all in stdlib and current; AES-CBC not deprecated (only OFB/CFB); zero external deps achievable
- (decision) AD-7 amendment: negotiation lives in a composite NegotiatingTransport inside internal/transport that implements Transport interface; Plug holds one Transport reference and calls Login once; composite tries KLAP then legacy internally and caches winner; Option override swaps which concrete transport is used; closes adversarial finding re negotiation ownership
